# AI開発知見の管理方針

## 検討日
2026-04-09

## 課題
- CLAUDE.md, skills, rules, settings等にAI関連設定が散らばる
- 個人の好みとチーム共有ルールの分離が必要
- worktreeでの並列実装時にも個人設定が有効であってほしい

## 採用方針: docs集約 + レイヤー分離

### ディレクトリ構成

```
docs/ai/
  shared/                  # チーム共有ルール（git管理）
    coding-rules.md
    review-guidelines.md
    architecture.md
  personal/                # 個人ルール（git管理 = 知見共有）
    nishimoto/
      rules.md
    matsuoka/
      rules.md

CLAUDE.md                  # チーム共有設定（docs/ai/shared/ を参照）
CLAUDE.local.md            # 個人設定（gitignore、自分のpersonal/を参照するポインタ）
```

### レイヤーの役割

| レイヤー | 置き場所 | git管理 | 用途 |
|---|---|---|---|
| チーム共有 | `docs/ai/shared/` | はい | コーディング規約、レビュー指針等 |
| 個人知見 | `docs/ai/personal/{name}/` | はい | 個人のルール・好み（他人も参照可） |
| 個人ポインタ | `CLAUDE.local.md` | いいえ | 自分のpersonalディレクトリを参照する1行 |
| 全プロジェクト共通 | `~/.claude/CLAUDE.md` | - | ユーザースコープの個人設定 |

### CLAUDE.local.md の中身（例）

```md
docs/ai/personal/nishimoto/ 配下のルールに従うこと
```

### 設計判断のポイント

#### なぜ `docs/ai/personal/` をgit管理するか
- 他人の設定が見れる → 知見共有になる
- PRで議論できる
- gitignoreだと知見が個人に閉じてしまう

#### なぜ `CLAUDE.local.md` をgitignoreにするか
- 個人ポインタがコンフリクトしない
- どのpersonalディレクトリを使うかは個人の選択

#### worktree対応
- `CLAUDE.local.md` はgitignoreなのでworktreeにコピーされない
- maiスキルのStep 4a（worktree作成時）で自動コピーする

```bash
# worktree作成後にCLAUDE.local.mdをコピー
cp {リポジトリパス}/CLAUDE.local.md /tmp/mai-{ブランチ名}/ 2>/dev/null || true
```

#### rulesとの使い分け
- rulesはglobsでスコープ指示ができるが、コンテキスト節約にはならない（検証済み）
- ファイル種別ごとのルールはrules、全体方針はCLAUDE.md + docs/ai/ が適切

### 参照の仕組み
- CLAUDE.md内で `@` インポートは未サポート
- 「docs/ai/shared/coding-rules.md を読んで従え」と自然言語で記述すればClaudeは読みに行く
- Cursor rules（`.cursor/rules/*.mdc`）は `@` でファイル参照可能
