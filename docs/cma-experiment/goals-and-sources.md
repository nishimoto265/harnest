# 目的・対象・情報源

## 1. 目的

### (a) Harness を自律的に作成していく
Harness（ AI コーディングエージェントに与えるルール群 ）を、人手で書き続けるのではなく、自律的に作成・更新していく仕組みを作る。

### (b) 個人の知見を共有・形式知化する
特定の個人に留まっている知見を、組織全体で参照できる形で形式知化する。

## 2. Harness の内容（ルール化の対象）

PR レビューやプロンプト等で、**何度も繰り返し指摘されている事項**をルール化していく。
ゴールは「同じ指摘を何度も繰り返さなくて済む状態」。

## 3. 情報源

### (a) GitHub の PR
- PR の内容
- レビューコメント

### (b) Databricks 上のプロンプト
各チームのメンバーが作成したものを参考にしてルールを作成する。

## 4. 今後の検討事項（一旦保留）

Databricks 上の情報だけでは、各プロンプトが **どの PR / リポジトリ / プロジェクトに紐付いているか** を追えない可能性がある。
将来的にはプラグイン等を導入し、ログを自動収集する仕組みを整えるのが理想。
